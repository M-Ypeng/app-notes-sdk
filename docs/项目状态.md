# 项目状态：已完成与待办

本文档用于集中记录 `@company/app-notes-sdk` 当前建设进度，便于开发者和 AI 编程工具快速判断哪些能力已经落地、哪些能力仍需继续建设。

更新时间：2026-05-25

## 已完成

### 产品能力

- 基础悬浮工具栏：支持新增标注、打开备注面板、显示/隐藏气泡。
- 工具栏拖拽：支持位置拖动和本地会话持久化。
- 元素选区：支持 hover 高亮、点击选择目标元素、Esc 取消和排除 SDK 自身元素。
- 备注表单：支持内容输入、标签选择、角色选择、图片上传、拖拽上传、粘贴截图和区域截图附加。
- AI 补充字段：支持期望结果、实际结果、复现步骤和修复线索。
- 页面气泡：支持当前页备注气泡渲染、点击打开详情、滚动/resize 后重定位。
- 低置信锚点保护：`low`、`invalid`、`rebind_required` 默认不渲染页面气泡，避免误导定位。
- 备注面板：支持当前页/其他页面分类、标签筛选、详情查看、定位、归档和重开。
- 锚点健康度展示：面板已展示稳定、一般、低置信、失效和需重绑统计。
- 重新绑定：支持从面板触发重新绑定，并持久化新的 anchor。
- 跨页定位：支持 `onNavigateToPage` 和 pending locate，页面切换后可继续定位目标备注。
- 修复记录：面板已支持记录修复摘要、改动文件、验证状态和验证时间。

### 技术能力

- Web Components 基础架构：client UI 基于 Custom Elements 和 Shadow DOM 实现。
- 数据源抽象：支持 memory 模式和 server 模式。
- 本地 Server：支持健康检查、备注列表、单条备注、追加备注、归档/重开、锚点更新、修复记录更新、图片上传和图片访问。
- 本地持久化：备注写入 `.app_notes/*.notes.json`，图片写入 `.app_notes/assets/`。
- 追加式写入：新评论追加到 `comments[]`，不覆盖历史评论。
- 运行上下文采集：备注创建时记录 URL、页面标题、视口、devicePixelRatio、userAgent 和 capturedAt。
- SPA 路由同步：支持监听 `history.pushState`、`replaceState`、`popstate` 和 `hashchange`。
- DOM 稳定等待：支持 SPA 页面切换后的延迟锚点重解析。
- 锚点生成与校验：支持 `data-note-id`、稳定 id、CSS selector、XPath、文本提示和布局线索。
- 锚点 evidence 写回：支持记录匹配方式、匹配分、失败原因和验证时间。
- AI 上下文导出：已提供 `pnpm export:ai-context`，输出 `.app_notes/AI_CONTEXT.md`。
- 宿主项目导出命令：server 包已提供 `app-notes-export-ai-context --root .`，宿主项目可通过 `pnpm notes:export` 生成 `.app_notes/AI_CONTEXT.md`。
- Server 文件缓存：读取 notes 文件时已按文件 mtime 缓存 parsed JSON，并在写入后更新缓存。
- Release 打包：已提供 `pnpm pack:release` 生成 client/server tarball；宿主项目接入说明维护在根目录 `HOST_USAGE.md`。
- 宿主项目接入验证：已在 React + Vite 宿主项目中验证 client 接入、`pnpm notes` 和 `pnpm notes:export`。

## 待办

### 产品待办

- 完成 Vue/原生 JS 真实宿主项目接入验证，并沉淀最小接入示例。
- 提供 React wrapper 和 Vue wrapper，降低业务项目接入成本。
- 增强锚点健康度报告：提供低置信锚点列表、建议添加的 `data-note-id` 和批量处理入口。
- 增强重新绑定体验：支持从失效备注直接进入重新绑定流程，并展示绑定前后差异。
- 增强截图能力：优化区域截图兼容性和失败降级提示。
- 增强图片预览：支持更完整的附件查看、删除确认和多图浏览。
- 增强导出能力：支持按页面、标签、角色、状态过滤导出。
- 提供 Prompt export：一键复制面向 Cursor、Codex、Claude Code、Roo Code 的修复提示。
- 增加修复结果回写流程：AI 修复后可批量写回修复摘要、改动文件和验证状态。
- 完善宿主项目集成指南：继续补充 Vue、原生 JS 和更多 Vite file 依赖场景。
- 完善稳定锚点命名规范：说明 `data-note-id` 在列表、表单、弹窗和动态内容中的推荐命名方式。

### 技术待办

- 补充单元测试：重点覆盖 `dom-anchor`、`page-path`、`storage`、`routes` 和 `export-ai-context`。
- 补充端到端测试：覆盖新增备注、图片上传、气泡定位、跨页定位、归档、重绑和导出。
- 增加气泡渲染优化：大量备注时只渲染当前视口附近或当前交互相关气泡。
- 提供 Export API：通过 `GET /api/export` 输出与 Markdown 导出等价的结构化上下文。
- 设计 MCP Server：支持 `list_annotations`、`get_annotation_context`、`mark_annotation_resolved`、`list_invalid_anchors`。
- 增加 schema migration：支持旧版本 notes 文件读取、升级和兼容提示。
- 增强冲突防御：多请求并发写入时增加更明确的文件级队列和错误恢复。
- 增强安全边界：限制 asset 文件名、访问路径和上传大小，补充异常用例。
- 增加性能压测：验证 100 个 notes 文件和单页 30 条 open 备注下的响应与滚动表现。
