# @company/app-notes-sdk 产品需求文档 PRD

## 1. 产品定位

`@company/app-notes-sdk` 是一个面向本地前端开发环境的视觉标注工具包。

它不是传统线上用户反馈系统，也不是普通截图批注工具，而是 **AI Coding Workflow 的前置标注层**：

> 让团队成员在可交互原型或本地开发页面上直接指出问题，并将这些问题沉淀为持久化、可共享、AI 可检索的视觉问题索引。

核心目标：

- 人可以在页面上准确指出业务逻辑、视觉规范、边界状态等问题。
- 标注数据持久化到项目目录，跨会话、跨工具保留。
- AI Coding 工具可以快速检索问题位置、页面路径、元素锚点和修复意图。
- 开发者启动项目后，可以重新在页面上复原标注并完成修复闭环。

## 2. 背景与问题

当前 AI Coding 工具已经具备截图、浏览器联动、圈选标注等能力，但多数能力是会话级、临时性的。

现有痛点：

- 页面问题散落在聊天记录、截图、Excel、飞书、Issue 中。
- AI 能看到截图，但不一定知道问题对应的页面、元素和组件锚点。
- AI 对 UI 问题的定位经常依赖自然语言猜测，容易改错位置。
- 一次会话里的标注无法沉淀成项目资产，后续无法批量检索。
- 团队协作中，PM/UI/QA/FE 对同一个页面问题缺少轻量、结构化的本地载体。

因此，本产品的核心价值不是“写备注”，而是：

> 把页面视觉问题转化为 AI 能检索和修复的结构化上下文。

## 3. 目标用户

- 前端开发：在本地环境快速定位并修复 UI/交互问题。
- UI 设计师：标注视觉规范、间距、颜色、字号、状态不一致问题。
- PM：标注业务逻辑、文案、边界流程问题。
- QA：标注验收问题、异常状态、边界测试问题。
- AI Coding 工具使用者：将页面问题批量交给 Cursor、Codex、Claude Code、Roo Code 等工具修复。

## 4. 使用场景

### 4.1 本地开发阶段视觉标注

用户启动业务项目和本地备注服务，在页面右下角打开备注工具，进入选区模式，点击目标元素后填写备注。

适用问题：

- 按钮颜色不符合规范。
- 文案需要调整。
- 某个状态缺少空态/加载态。
- 表格字段展示不完整。
- 交互逻辑需要补充说明。

### 4.2 AI 修复前的问题检索

AI Coding 工具读取项目内标注数据，理解：

- 哪些页面存在 open 状态的问题。
- 每条问题对应哪个元素。
- 用户希望修改什么。
- 该问题是否包含截图、页面路径、元素文本和定位线索。

### 4.3 修复后视觉复核

开发者或 AI 修改代码后，重新启动页面。SDK 根据项目内标注数据恢复视觉标注，用户可以定位元素、确认效果、归档或重新打开问题。

### 4.4 项目内共享

标注结果作为项目资产保存，可通过以下方式共享：

- 随代码分支提交。
- 单独打包给开发者。
- 导出 JSON/Markdown 报告。
- 由 AI 工具或 MCP 读取。

注意：新增视觉标注和页面复核需要项目启动；但问题数据本身可以在不启动项目时被 AI 或开发者读取。

## 5. 产品边界

### 5.1 产品要做

- 本地开发环境可用的页面视觉标注。
- 跨 React/Vue/原生项目的接入能力。
- 项目内持久化问题索引。
- 页面元素锚点解析与定位。
- 备注列表、详情、视觉提示、归档。
- 图片上传和粘贴截图。
- AI 可读的结构化备注数据。

### 5.2 产品暂不做

- 不做完整 SaaS 协作平台。
- 不替代 Jira、Linear、GitHub Issues。
- 不面向生产环境用户反馈。
- 不做复杂权限、账号、云端同步。
- 不在浏览器端直接写本地文件。

## 6. 核心产品约束

- **本地优先**：数据默认存放在当前项目中，不依赖云端服务。
- **开发环境优先**：工具用于开发、联调、验收阶段，不应进入生产产物。
- **跨框架可用**：应能接入 React、Vue、Angular、原生 JS 等不同技术栈。
- **AI 友好**：每条标注都应尽量包含 AI 定位和修复所需的结构化上下文。
- **不误导**：低置信度锚点不得错误展示，宁可提示失效，也不要指向错误元素。
- **锚点优先级明确**：`data-note-id` 是唯一强推荐锚点；XPath 和普通 CSS selector 只能作为回退手段。

## 7. 功能需求

### 7.1 悬浮入口

功能：

- 页面右下角展示可拖拽悬浮球。
- 点击悬浮球展开备注面板。
- 快捷键 `Ctrl+Shift+N` 打开或关闭备注面板。
- 面板支持一键隐藏/显示所有备注提示。

交互要求：

- 不遮挡主页面核心操作。
- 悬浮球拖拽后位置在当前会话中保持。
- 样式与宿主页面隔离。

### 7.2 备注面板

面板展示：

- 当前页面备注列表。
- 后续扩展为全部页面备注列表。
- 备注摘要、标签、角色、状态、页面路径。
- 当前页备注显示“定位元素”。
- 非当前页备注显示“前往页面”。

详情展示：

- 备注内容。
- 标签、角色、状态。
- 图片附件。
- 锚点信息。
- 锚点匹配证据，例如 matchedBy、matchScore、失败原因、最近验证时间。
- AI 补充信息，例如期望、实际、复现步骤、修复线索。
- 创建时间。
- 归档/重新打开。
- 定位元素。
- 重新绑定元素。

### 7.3 新增备注选区

用户点击“新增备注”后进入选区模式。

选区模式要求：

- 鼠标 hover 时高亮目标元素。
- 显示当前目标元素信息，例如 tag、id、class、`data-note-id`。
- 点击元素后打开备注表单。
- 按 Esc 取消。
- 工具自身元素不可被标注。

锚点规则：

1. 优先读取目标元素或最近父级元素的 `data-note-id`。
2. 若存在 `data-note-id`，以其作为稳定锚点。
3. 若不存在，使用可回退定位策略。
4. 同时保存文本摘要、元素类型等辅助信息。

### 7.4 备注输入表单

字段：

- 文本内容。
- 标签：
  - `[疑问]`
  - `[变更建议]`
  - `[逻辑补充]`
  - `[视觉规范]`
- 备注人角色：
  - PM
  - UI
  - FE
  - BE
  - QA
- 图片附件。
- AI 补充信息（默认折叠）：
  - 期望结果。
  - 实际结果。
  - 复现步骤。
  - 修复线索。

图片能力：

- 点击上传。
- 拖拽上传。
- 粘贴截图 - 直接复制快捷输入。

校验：

- 文本和图片至少有一项。
- 锚点必须存在。

### 7.5 备注展示与定位

页面上应渲染视觉提示：

- 目标元素附近展示备注气泡、pin 或连接线（强指引）。
- 气泡展示摘要、标签、角色。
- 点击气泡进入详情面板。
- 支持隐藏/显示所有备注提示。

定位要求：

- 优先使用稳定锚点。
- 页面结构变化后，应通过辅助信息校验锚点是否仍可信。
- 低置信度锚点不得错误展示。
- 找不到可信元素时，应提示失效或支持重新绑定。
- 备注详情和列表应能体现锚点健康状态，例如稳定、低置信、失效、待重新绑定。

### 7.6 跨页备注

备注数据应包含页面路径。

交互：

- 当前页备注可以直接定位。
- 非当前页备注显示“前往页面”。
- 页面切换后自动打开备注详情并定位元素。
- 若无法自动切换路由，应允许宿主项目接入自定义导航函数。

### 7.7 追加式备注与状态流转

新增备注后，系统应追加写入，不覆盖历史评论。

状态流转：

- `open`：待处理。
- `archived`：已归档。

约束：

- 不提供历史内容编辑。
- 支持关闭/归档和重新打开。
- 后续可扩展 `resolved`、`invalid`、`rebind_required` 等状态。

### 7.8 锚点健康度报告

已提供基础锚点健康度报告，帮助用户主动发现问题，而不是等定位失败时才发现。

报告内容：

- 当前页面锚点总数。
- 稳定锚点数量，例如基于 `data-note-id`。
- 低置信锚点数量，例如仅依赖 XPath 或动态 CSS selector。
- 已失效锚点数量。
- 需要重新绑定的锚点列表。

交互要求：

- 在备注面板中展示健康度入口。
- 对低置信锚点给出增加 `data-note-id` 的建议。
- 对失效锚点提供“重新绑定元素”操作。

### 7.9 AI 上下文导出

应提供本地导出能力，将 `.app_notes/*.notes.json` 汇总为 AI 可直接读取的 Markdown 文件。

导出结果：

- 输出到 `.app_notes/AI_CONTEXT.md`。
- 按 `invalid / rebind_required / low` 优先排序。
- 包含页面、锚点、备注内容、标签、角色、图片路径。
- 包含锚点健康度、匹配方式、匹配分、失败原因。
- 包含运行上下文，例如 URL、标题、视口尺寸、设备像素比。
- 包含用户补充的期望、实际、复现步骤、修复线索。
- 包含修复记录，例如改动文件、修复摘要、验证状态。

### 7.10 项目落地明细

SDK 在宿主项目中的落地分为“标注采集”和“AI 修复”两个闭环。

标注采集明细：

- 开发者在宿主项目的开发环境中按需加载 `@company/app-notes-client`。
- 本地启动 `@company/app-notes-server`，并将 `--root` 指向宿主项目根目录。
- PM/UI/QA/FE 在运行页面中通过悬浮工具栏进入选区模式。
- 选中目标元素后填写备注内容、标签、角色和图片附件。
- Client 将锚点、备注、运行上下文和附件通过 HTTP API 交给 Server。
- Server 将结构化数据写入宿主项目 `.app_notes/*.notes.json`，图片写入 `.app_notes/assets/`。
- 页面刷新或项目重启后，SDK 根据 `.app_notes` 恢复当前页可定位备注气泡。
- 对低置信、失效或需要重新绑定的锚点，面板中应明确提示，不应误渲染到错误元素。

AI 修复明细：

- AI Coding 工具优先读取 `.app_notes/AI_CONTEXT.md`，必要时回查原始 `.notes.json`。
- AI 按 `invalid / rebind_required / low / medium / stable` 风险顺序理解问题。
- 对每条 open 备注，AI 需要同时参考页面路径、锚点、文本提示、图片、角色、标签和 AI 补充字段。
- 修复时只修改与备注目标相关的 UI、交互或逻辑，避免扩大改动范围。
- 修复后可通过 `fix.summary`、`fix.changedFiles`、`fix.verified`、`fix.verifiedAt` 写回修复记录。
- 人工复核通过后，将对应 comment 归档；若锚点失效，应先重新绑定再继续处理。

## 8. AI Coding Workflow 能力

### 8.1 AI 可读上下文

每条备注应尽量沉淀以下信息：

- 页面路径。
- 用户备注内容。
- 角色和标签。
- 元素定位线索。
- 元素文本摘要。
- 图片附件。
- 创建时间和状态。
- 运行环境：完整 URL、页面标题、视口尺寸、devicePixelRatio、userAgent。
- 锚点证据：matchedBy、matchScore、failureReason、lastValidatedAt。
- AI 补充字段：expected、actual、stepsToReproduce、fixHints。
- 修复记录：summary、changedFiles、verified、verifiedAt。

### 8.2 AI 检索目标

AI Coding 工具读取标注后，应能回答：

- 当前项目有哪些待修复视觉问题？
- 每个问题出现在什么页面？
- 用户希望改什么？
- 问题大概率对应哪个元素？
- 哪些问题已经失效，需要人工重新绑定？
- 哪些问题已经有修复记录，哪些仍未验证？

### 8.3 Prompt 场景

后续可支持复制 Prompt：

```text
请读取当前项目中的所有 open 状态视觉标注，
根据页面路径、元素锚点和备注内容修复对应 UI 问题。
修复时不要改动无关逻辑。
```

## 9. 竞品与差异化

传统工具：

- BugHerd
- Marker.io
- Userback
- Pastel
- Ruttl

这些工具更偏线上反馈、客户协作、任务流转。

AI 工具内置标注更偏临时会话上下文。

本产品差异化：

- 面向本地开发环境。
- 标注数据项目内持久化。
- 可共享、可复原。
- 面向 AI Coding 工具结构化输出。
- 优先使用项目级稳定锚点。

一句话差异化：

> 不是让人更方便地写反馈，而是让 AI 更快、更准地找到要修的问题。

## 10. 成功指标

- AI 定位问题所需上下文更少。
- UI 问题从标注到修复的时间缩短。
- AI 改错元素的概率降低。
- open 状态标注可以被批量读取和处理。
- 历史标注可以在项目重启后复原。
- 失效锚点可以被识别，而不是错误显示。
- 低置信锚点占比可被统计，并随项目接入 `data-note-id` 逐步下降。

- 悬浮入口。
- 选区标注。
- 备注表单。
- 图片附件。

- 稳定锚点策略。

- 跨页跳转定位。

- Prompt export。

- 修复记录写回与导出。

### M4: AI 工具集成

- MCP Server。
- AI 工具读取标注。
- 标记修复结果。

## 12. 当前状态

### 12.1 已完成

- 基础悬浮工具栏：支持新增标注、打开备注面板、显示/隐藏气泡。
- 工具栏拖拽：支持位置拖动和本地会话持久化。
- 元素选区：支持 hover 高亮、点击选择目标元素、Esc 取消和排除 SDK 自身元素。
- 备注表单：支持内容输入、标签选择、角色选择、图片上传、拖拽上传、粘贴截图和区域截图附加。
- AI 补充字段：备注表单已支持期望结果、实际结果、复现步骤和修复线索。
- 页面气泡：支持当前页备注气泡渲染、点击打开详情、滚动/resize 后重定位。
- 低置信锚点保护：`low`、`invalid`、`rebind_required` 默认不渲染页面气泡，避免误导定位。
- 备注面板：支持当前页/其他页面分类、标签筛选、详情查看、定位、归档和重开。
- 锚点健康度展示：面板已展示稳定、一般、低置信、失效和需重绑统计。
- 重新绑定：支持从面板触发重新绑定，并持久化新的 anchor。
- 跨页定位：支持 `onNavigateToPage` 和 pending locate，页面切换后可继续定位目标备注。
- SPA 路由同步：支持监听 `history.pushState`、`replaceState`、`popstate` 和 `hashchange`。
- DOM 稳定等待：支持 SPA 页面切换后的延迟锚点重解析。
- 数据源抽象：支持 memory 模式和 server 模式。
- 本地 Server：支持健康检查、备注列表、单条备注、追加备注、归档/重开、锚点更新、修复记录更新、图片上传和图片访问。
- 本地持久化：备注写入 `.app_notes/*.notes.json`，图片写入 `.app_notes/assets/`。
- 追加式写入：新评论追加到 `comments[]`，不覆盖历史评论。
- 运行上下文采集：备注创建时记录 URL、页面标题、视口、devicePixelRatio、userAgent 和 capturedAt。
- 修复记录：面板和 Server 已支持 `fix.summary`、`fix.changedFiles`、`fix.verified`、`fix.verifiedAt`。
- AI 上下文导出：已提供 `pnpm export:ai-context`，输出 `.app_notes/AI_CONTEXT.md`。
- Release 打包：已提供 `pnpm pack:release` 生成 client/server tarball。
